Materi 5: Mengambil Data dari Data Warehouse

Ini adalah bagian pertama dari pembangunan alur kerja Nathan. Di sini, kita akan belajar cara mendapatkan data dengan membuat permintaan HTTP (HTTP requests) menggunakan node HTTP Request.

Setelah menyelesaikan bagian ini, alur kerja kita akan terlihat seperti ini (kita akan menambahkan pemicu jadwal nanti, untuk saat ini kita mulai dengan trigger manual):

Langkah 1: Membuat Workflow Baru

Buka Editor UI n8n Anda dan buat workflow baru. Kita bisa menggunakan salah satu dari dua cara:

  • Pilih Ctrl+Alt+N (atau Cmd+Option+N di Mac).
  • Buka menu kiri, navigasi ke Workflows, dan pilih Add workflow.

(Ganti gambar ini) Membuat workflow baru dari sidebar.

Setelah kanvas baru terbuka, ganti nama workflow ini menjadi "Workflow Nathan".

Langkah 2: Menambahkan Node HTTP Request

Hal pertama yang harus kita lakukan adalah mengambil data dari data warehouse lama ABCorp.

Di Materi 2, kita menggunakan node khusus (Hacker News). Tapi bagaimana jika tidak ada node khusus untuk layanan yang kita butuhkan? Seperti kasus data warehouse lama Nathan.

Untungnya, Nathan memberi tahu kita bahwa data warehouse-nya memiliki API endpoint. Ini adalah "pintu" yang bisa kita gunakan untuk meminta data. Di n8n, node yang sempurna untuk ini adalah HTTP Request.

Apa itu Node HTTP Request?
Anggap saja node ini seperti browser mini. Kita memberinya URL, dan ia akan "mengunjungi" URL tersebut untuk mengambil data apa pun yang ada di sana. Ini adalah salah satu node paling serbaguna dan penting di n8n.

Di kanvas Anda, klik tanda + pada node "Start" Anda, lalu cari dan tambahkan node HTTP Request.

(Ganti gambar ini) Menambahkan node HTTP Request ke kanvas.

Jendela konfigurasi node akan terbuka.

Langkah 3: Memahami & Menyiapkan Kredensial

Node ini akan membutuhkan kredensial. Kredensial adalah "kunci" (seperti username/password atau API key) yang membuktikan bahwa kita berhak mengakses data.

Catatan Kredensial
Jika Anda mengikuti kursus resmi n8n, mereka akan memberikan kredensial demo di email pendaftaran. Untuk dokumentasi ini, kita akan berpura-pura telah menerima kredensial tersebut.

Langkah 4: Mengonfigurasi Node HTTP Request

Ini adalah bagian paling detail. Kita akan mengisi semua parameter yang dibutuhkan untuk "membuka pintu" API.

Bagian 1: URL dan Headers

Di tab Parameters, isi sebagai berikut:

  • Method: Pastikan ini diatur ke GET. (Artinya kita ingin "MENDAPATKAN" data).
  • URL: Masukkan Dataset URL yang Anda terima dari email kursus. (Contoh: https://internal.users.n8n.cloud/...)
  • Send Headers: Aktifkan (geser ke true). Headers adalah "metadata" tambahan yang kita kirim bersama permintaan kita.
  • Specify Headers: Pastikan Using Fields Below terpilih.
  • Header Parameters > Name: Masukkan unique_id.
  • Header Parameters > Value: Masukkan Unique ID yang Anda terima dari email.

(Ganti gambar ini) Screenshot konfigurasi URL dan Header Parameters.

Bagian 2: Otentikasi & Kredensial

Sekarang, scroll ke bawah ke bagian Authentication.

  • Authentication: Pilih Generic Credential Type.
  • Generic Auth Type: Pilih Header Auth. (Ini akan muncul setelah Anda memilih langkah sebelumnya).
  • Credential for Header Auth: Ini adalah kuncinya. Klik + Create new credential.

(Ganti gambar ini) Kita akan membuat kredensial baru untuk API ini.

Sebuah jendela popup "Create new credential" akan muncul. Di sinilah kita menyimpan "kunci" rahasia kita dengan aman.

  • Name: Masukkan Header Auth name dari email (Misal: Nathan's ABCorp data warehouse account).
  • Value: Masukkan Header Auth value dari email.

(Ganti gambar ini) Mengisi detail kredensial rahasia kita.

Klik tombol Save di jendela popup. n8n akan menyimpan kredensial ini dengan aman. Jendela popup akan tertutup, dan Anda akan melihat kredensial Anda sekarang sudah terpilih.

Ini adalah data yang Nathan butuhkan! Kita bisa melihat 30 item (pelanggan) dengan lima kolom:
