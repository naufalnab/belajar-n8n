Materi 3: Mengotomatiskan Kasus (Dunia Nyata)

Di materi sebelumnya, kita telah belajar dasar-dasar UI dan membuat workflow mini. Sekarang, kita akan naik level. Kita akan menerapkan pengetahuan kita pada skenario studi kasus yang realistis.

Rencana Alur Kerja (Workflow)

Dari percakapan di atas, kita bisa memecah masalah Nathan menjadi beberapa langkah logis yang bisa ditangani n8n:

  1. Pemicu (Trigger): Kita perlu sesuatu yang berjalan "setiap hari Senin". Ini akan menggunakan Schedule Node.
  2. Ambil Data: Kita perlu "mengambil data dari API". Kita akan menggunakan HTTP Request Node.
  3. Pisahkan Data: Kita harus memisahkan data "Processing" dan "Booked". Ini adalah pekerjaan sempurna untuk IF Node.
  4. Tugas "Booked":
    • Menghitung total (sum) dari pesanan "Booked".
    • Mengirim totalnya ke Discord (Discord Node).
  5. Tugas "Processing":
    • Membuat spreadsheet dari pesanan "Processing".
    • Kita bisa pakai Google Sheets Node. (Contoh di JSON menggunakan Airtable, yang fungsinya serupa).
