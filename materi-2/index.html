Materi 2: Modifikasi - Workflow RSS Feed Berita

Di sinilah kita benar-benar menerapkan prinsip Amati, Tiru, Modifikasi (ATM). Di dokumentasi resmi, mereka mencontohkan cara mengambil data dari Hacker News. Itu bagus, tapi mari kita buat sesuatu yang lebih relevan untuk kita.

Kita akan memodifikasi alur tersebut. Alih-alih Hacker News, kita akan menggunakan node yang jauh lebih kuat dan universal: RSS Feed Reader.

Tujuan kita: Membuat workflow yang secara otomatis mengambil berita teknologi terbaru langsung dari detikINET (Detik.com).

Konsepnya 100% sama dengan tutorial resmi, tapi hasilnya jauh lebih personal dan berguna:

  1. Memulai Workflow (Masih pakai Manual Trigger).
  2. Menambahkan Node Pengambil Data (Modifikasi: RSS Feed Reader).
  3. Mengonfigurasi Node (Mengisi URL feed, bukan 'Limit' atau 'Keyword').
  4. Mengeksekusi dan Memeriksa Hasil (Output-nya akan berbeda!).
  5. Menyimpan Workflow kita yang sudah dimodifikasi.

Langkah 1: Memulai Alur Kerja (Tiru)

Langkah pertama ini kita tiru persis. Setiap workflow butuh pemicu. Saat Anda membuat workflow baru, Anda akan disambut oleh kanvas kosong dengan node "Start".

(Ganti gambar ini) Kanvas kosong. Node "Start" ini adalah Manual Trigger bawaan kita.

Kita bisa langsung klik tanda + di node "Start" itu. Node ini sempurna untuk pengujian karena workflow hanya akan berjalan ketika kita menekan tombol "Execute workflow". Tidak perlu menghapus atau menambah node trigger baru.

Tips Belajar
Node "Start" dan node "Manual Trigger" (jika dicari di panel node) memiliki fungsi yang identik. Jangan bingung jika melihat keduanya, anggap saja mereka sama.

Langkah 2: Menambahkan Node RSS Feed (Modifikasi)

Inilah langkah "Modifikasi" kita. Alih-alih mencari "Hacker News", kita akan mencari node yang jauh lebih serbaguna.

Klik ikon + di sebelah kanan node "Start" Anda. Ini akan membuka panel node.

Di kotak pencarian, ketik: "RSS". Anda akan langsung melihat node "RSS Feed Reader". Klik node tersebut.

(Ganti gambar ini) Panel node setelah saya mencari 'RSS'. Node inilah yang kita inginkan.

Berbeda dengan node Hacker News yang mungkin menanyakan "Action", node RSS Feed Reader ini lebih simpel. Ia hanya punya satu tujuan: membaca feed. Jadi, node ini akan langsung ditambahkan ke kanvas dan jendela pengaturannya akan terbuka secara otomatis.

Langkah 3: Mengonfigurasi Node RSS Feed

Jendela pengaturan node RSS Feed Reader jauh lebih sederhana daripada Hacker News, yang justru bagus untuk dipelajari. Ini menunjukkan bahwa setiap node itu unik.

Parameters vs. Settings (Analogi Resep)
Konsep ini tetap sama. Parameters adalah *bahan* resep (spesifik untuk node RSS). Settings adalah *catatan di pinggir resep* (sama untuk semua node).

Bagian 1: Mengatur Parameters (Bahan Utama)

Node ini hanya punya satu parameter *wajib*: URL.

Kita perlu memberinya link ke file RSS feed berita. Untuk contoh ini, kita akan memakai feed dari detikINET.

  • Di bidang URL, masukkan: https://rss.detik.com/inet.xml

(Ganti gambar ini) Satu-satunya hal yang saya isi. Sangat sederhana!

Bagaimana Cara Menemukan URL RSS Feed Situs Lain?
Hampir semua situs berita/blog memilikinya! Coba salah satu cara ini:
  1. Cari halaman di situs mereka yang berjudul "RSS" atau "Feed".
  2. Buka situsnya, klik kanan, pilih "View Page Source" (Lihat Sumber Halaman), lalu cari (Ctrl+F) kata type="application/rss+xml". Link di dalam href itulah URL-nya.
Contoh lain: https://www.theverge.com/rss/index.xml

Bagian 2: Mengatur Settings (Catatan Resep)

Langkah ini 100% sama dengan tutorial resmi, tapi kita akan ubah catatannya. Klik tab "Settings".

  • Notes: Kita tulis catatan yang jelas, misal: Mengambil 10 berita terbaru dari detikINET.
  • Display note in flow?: Aktifkan (geser ke kanan).

Terakhir, mari kita ganti nama node ini agar profesional. Klik dua kali pada nama "RSS Feed Reader" di kiri atas, dan ganti namanya menjadi "Ambil Berita detikINET", lalu klik "Rename".

(Ganti gambar ini) Saya mengisi Notes dan mengganti nama node agar mudah dibaca.

Langkah 4: Mengeksekusi & Mengamati Hasilnya

Resep kita sudah siap. Saatnya menguji node ini. Di jendela node "Ambil Berita detikINET", klik tombol "Execute step" (ikon 'Play' ▶️) di pojok kanan atas.

Setelah beberapa detik, node akan selesai bekerja dan menampilkan tab "Output". Jika Anda kembali ke kanvas (klik "Back to canvas"), Anda akan melihat tanda centang hijau di atas node Anda. Sukses!

(Ganti gambar ini) Tanda centang hijau di node RSS kita.

Memahami Output RSS (Ini Berbeda!)

Output dari node RSS akan memiliki struktur yang berbeda dari node Hacker News. Inilah gunanya memodifikasi! Kita jadi belajar hal baru.

Buka kembali jendelanya dan lihat tab Table. Anda akan melihat kolom-kolom seperti title, link, pubDate (tanggal publikasi), creator, dan content. Ini adalah struktur standar RSS.

(Ganti gambar ini) Tampilan 'Table'. Kita bisa lihat semua judul berita terbaru dari detikINET!

Sekarang, klik tab JSON. Anda akan melihat data mentahnya. Ini sangat berguna untuk melihat data apa *sebenarnya* yang kita dapatkan.

(Ganti gambar ini) Tampilan 'JSON'. Perhatikan, ada data seperti 'content:encoded' yang berisi HTML lengkap dari artikelnya!

(Ganti gambar ini) Tampilan 'Schema'. Ini seperti daftar isi data, menunjukkan 'title' adalah string, 'pubDate' adalah string, dll.

Observasi Penting: Di Mana Filternya?
Anda mungkin sadar: "Lho, kok hasilnya ada 20?" atau "Bagaimana cara saya membatasi (limit) jadi 10?".

Selamat! Anda baru saja menemukan poin belajar yang penting. Node RSS Feed Reader tidak punya parameter 'Limit' bawaan seperti node Hacker News. Node ini dirancang untuk mengambil *semua* yang ada di file RSS.

Di materi-materi selanjutnya, kita akan belajar cara memanipulasi output ini menggunakan node lain (seperti Split in Batches atau Code) untuk mengambil hanya 10 item teratas. Tapi untuk sekarang, menerima semua datanya sudah merupakan sebuah kesuksesan!

Langkah 5: Menyimpan Workflow Modifikasi Kita

Langkah terakhir ini kita tiru lagi. Klik "Back to canvas".

Klik nama workflow "My workflow" di bagian atas layar, dan beri nama yang bagus, seperti "Materi 2 - Ambil Berita detikINET".

(Ganti gambar ini) Memberi nama yang jelas pada workflow kita.

Terakhir, tekan Ctrl + S (atau Cmd + S) atau klik tombol "Save" di pojok kanan atas sampai berubah menjadi teks "Saved" berwarna abu-abu.

Ringkasan: Kita Telah Berhasil Memodifikasi!

Luar biasa! Anda tidak hanya "meniru" dokumentasi, tapi Anda telah berhasil mengamati konsepnya dan memodifikasinya untuk kebutuhan yang lebih nyata dan relevan.

Di materi ini, kita belajar:

  • Bahwa node RSS Feed Reader adalah alternatif yang hebat untuk mengambil data berita.
  • Cara mencari URL RSS feed dari sebuah situs.
  • Bahwa setiap node memiliki Parameter unik (RSS hanya butuh URL).
  • Cara membaca output Table dan JSON yang strukturnya berbeda (khas RSS).
  • Observasi penting bahwa tidak semua node punya parameter 'Limit', dan itu wajar.

Di materi selanjutnya, kita akan belajar cara mengambil output ini dan *melakukan sesuatu* dengannya, seperti memfilter, memanipulasi, dan mengirimkannya ke layanan lain.
