Materi 2: Membangun Workflow Mini

Di materi ini, kita akan membangun sebuah workflow kecil yang mengambil 10 artikel tentang "automasi" dari Hacker News. Prosesnya terdiri dari lima langkah:

  1. Menambahkan node Manual Trigger
  2. Menambahkan node Hacker News
  3. Mengonfigurasi node Hacker News
  4. Mengeksekusi node
  5. Menyimpan workflow

1. Menambahkan Node Manual Trigger

Buka panel node (ingat: Anda bisa membukanya dengan memilih ikon + di pojok kanan atas kanvas atau menekan tombol Tab di keyboard Anda).

Lalu:

  • Cari node Manual Trigger.
  • Pilih node tersebut saat muncul di pencarian.

Ini akan menambahkan node Manual Trigger ke kanvas Anda, yang memungkinkan Anda menjalankan workflow kapan saja dengan memilih tombol Execute workflow.

(Ganti gambar ini) Menambahkan node Manual Trigger ke kanvas.

Tips Pembuatan Workflow Cepat
Anda bisa melewati langkah ini di masa depan. Menambahkan node lain tanpa trigger akan otomatis menambahkan node Manual Trigger ke workflow.
Skenario Dunia Nyata
Dalam skenario dunia nyata, Anda mungkin ingin mengatur jadwal (schedule) atau trigger lain untuk menjalankan workflow.

2. Menambahkan Node Hacker News

Pilih ikon + di sebelah kanan node Manual Trigger untuk membuka panel node.

Lalu:

  • Cari node Hacker News.
  • Pilih node tersebut saat muncul di pencarian.
  • Di bagian Actions, pilih Get many items.

n8n akan menambahkan node ke kanvas Anda dan jendela node akan terbuka untuk menampilkan detail konfigurasinya.

3. Mengonfigurasi Node Hacker News

Saat Anda menambahkan node baru ke Editor UI, node tersebut otomatis aktif. Detail node akan terbuka di jendela dengan beberapa opsi:

  • Parameters: Menyesuaikan parameter untuk menyempurnakan dan mengontrol fungsionalitas node.
  • Settings: Menyesuaikan pengaturan untuk mengontrol desain dan eksekusi node.
  • Docs: Membuka dokumentasi n8n untuk node ini di jendela baru.
Parameters vs. Settings
Parameters berbeda untuk setiap node, tergantung pada fungsinya.
Settings sama untuk semua node.

Parameters

Kita perlu mengonfigurasi beberapa parameter agar node Hacker News berfungsi:

  • Resource: All
    Resource ini memilih semua catatan data (artikel).
  • Operation: Get Many
    Operasi ini mengambil semua artikel yang dipilih.
  • Limit: 10
    Parameter ini menetapkan batas jumlah hasil yang dikembalikan oleh operasi Get Many.
  • Additional Fields > Add Field > Keyword: automation
    Ini adalah opsi yang bisa Anda tambahkan untuk membuat permintaan Anda lebih spesifik. Untuk contoh ini, kita hanya ingin mendapatkan artikel yang menyertakan kata kunci "automasi" (automation).

Settings

Bagian Settings mencakup beberapa opsi untuk desain dan eksekusi node. Dalam kasus ini, kita hanya akan mengonfigurasi dua pengaturan terakhir, yang mengatur tampilan node di kanvas Editor UI.

Di dalam Settings node Hacker News, edit:

  • Notes: Get the 10 latest articles.
  • Display note in flow?: (toggle ke true)
Seringkali sangat membantu untuk menambahkan deskripsi singkat di node tentang apa yang dilakukannya. Ini sangat berguna untuk workflow yang kompleks atau yang dibagikan!

Opsi ini akan menampilkan Catatan (Note) di bawah node di kanvas. Konfigurasi settings sekarang akan terlihat seperti ini:

Mengganti Nama Node
Anda dapat mengganti nama node dengan nama yang lebih deskriptif. Ada tiga cara untuk melakukannya:
  • Pilih node yang ingin Anda ganti namanya dan tekan tombol F2.
  • Klik dua kali pada node untuk membuka jendela node. Klik pada nama node di sudut kiri atas jendela, ganti namanya, lalu klik Rename.
  • Klik kanan pada node dan pilih opsi Rename.

4. Mengeksekusi Node

Pilih tombol Execute step di jendela detail node. Anda akan melihat 10 hasil di tampilan Output Table.

(Ganti gambar ini) Hasil node Hacker News dalam tampilan Tabel.

Eksekusi Node

Eksekusi Node
Sebuah eksekusi node mewakili satu kali jalannya node tersebut untuk mengambil atau memproses data yang ditentukan.

Jika node dieksekusi dengan sukses, tanda centang hijau kecil muncul di atas node di kanvas.

(Ganti gambar ini) Tanda centang hijau menunjukkan eksekusi berhasil.

Jika tidak ada masalah dengan parameter, data yang diminta akan tampil di jendela node dalam format Table, JSON, dan Schema. Anda dapat berpindah antar tampilan ini.

Tampilan Table vs JSON
Tampilan Table adalah default. Ini menampilkan data dalam tabel. Tampilan JSON menampilkan output data mentah dalam format JSON.

(Ganti gambar ini) Hasil node Hacker News dalam tampilan JSON.

Jendela node menampilkan lebih banyak informasi tentang eksekusi node:

  • Di sebelah judul Output, ada ikon info. Jika Anda mengarahkan kursor ke sana, Anda akan mendapatkan dua info lagi: Start Time (Kapan eksekusi dimulai) dan Execution Time (Berapa lama waktu yang dibutuhkan).
  • Tepat di bawah judul Output, Anda akan melihat info: 10 items. Ini menampilkan jumlah item (catatan) yang dikembalikan.
Error pada Node
Ikon peringatan merah pada node berarti node tersebut mengalami error. Ini mungkin terjadi jika kredensial node hilang/salah atau parameter tidak dikonfigurasi dengan benar.

(Ganti gambar ini) Node yang menunjukkan error.

5. Menyimpan Workflow

Setelah Anda selesai mengedit node, pilih Back to canvas untuk kembali ke kanvas utama.

Secara default, workflow Anda disimpan sebagai "My workflow." Untuk materi ini, ganti nama workflow menjadi "Hacker News workflow."

Pengingat: Anda dapat mengganti nama workflow dengan mengklik nama workflow di bagian atas Editor UI.

Setelah Anda mengganti namanya, pastikan untuk menyimpannya. Ada dua cara untuk menyimpan workflow:

  • Tekan Ctrl + S (atau Cmd + S untuk Mac) di keyboard Anda.
  • Pilih tombol Save di pojok kanan atas Editor UI.

Jika Anda melihat teks Saved berwarna abu-abu, berarti workflow Anda sudah tersimpan secara otomatis.
