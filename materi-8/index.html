Node "Edit Fields (Set)"

Untuk tugas ini, kita akan menggunakan node Edit Fields (Set). Anggap saja node ini sebagai "penjaga gerbang" data. Node ini bisa memilih data mana yang boleh lewat, memodifikasi data, atau bahkan membuat data baru.

Langkah 1: Menyisipkan Node "Edit Fields"

Kita perlu menambahkan node ini di antara IF Node dan Airtable Node.

  1. Hapus koneksi (panah) antara IF (true) dan Airtable (klik ikon tempat sampah di panah).
  2. Seret (drag) node Airtable Kita sedikit ke kanan di kanvas agar ada ruang.
  3. Klik ikon + pada output "true" dari IF Node.

Kita akan menyisipkan node baru di jalur 'true'.

Sekarang cari node Edit Fields (Set) dan pilih.

Node 'Edit Fields (Set)' adalah yang kita butuhkan.

Langkah 2: Mengonfigurasi Node "Edit Fields"

Dengan jendela "Edit Fields" terbuka, konfigurasikan parameter berikut:

  • Mode: Pastikan diatur ke Manual Mapping.
  • Fields to Set: Di sinilah kita memilih data Kita. Kita bisa menggunakan Expression Editor seperti di Materi 7, tapi ayo kita coba cara drag-and-drop*:
    1. Di panel "INPUT" di sebelah kiri, cari orderID (dari node IF) dan seret (drag) ke kotak Name di sebelah kanan.
    2. Lakukan hal yang sama untuk employeeName.

Cara mudah memilih kolom: cukup seret dari kiri ke kanan.

  • Include Other Input Fields: Ini adalah langkah paling penting. Matikan (geser ke false).
Sangat Penting: "Include Other Input Fields"
Jika tombol ini AKTIF, n8n akan mengambil orderID, employeeName, DAN SEMUA KOLOM LAINNYA (customerID, orderPrice, dll.).
Dengan MEMATIKANNYA, kita memberi tahu n8n: "Saya HANYA ingin kolom yang saya pilih secara manual di atas."

Pastikan tombol ini MATI untuk memfilter kolom Kita.

Langkah 3: Menguji Node "Edit Fields"

Klik Execute step. Anda akan melihat hasilnya. Perhatikan: Input memiliki 5 kolom, tetapi Output sekarang hanya memiliki 2 kolom!

Kita berhasil memangkas data Kita! Hanya 2 kolom yang lolos.

Tutup jendela node "Edit Fields".

Langkah 4: Menyiapkan Tabel Airtable Baru

Data kita sekarang hanya memiliki 2 kolom, tetapi tabel "orders" lama kita di Airtable memiliki 5 kolom. Ini bisa menyebabkan error.

Praktik terbaiknya adalah membuat tabel baru yang bersih khusus untuk data ini.

  1. Buka 'Base' "beginner course" Kita di Airtable.
  2. Klik tanda + untuk "Add or import" dan pilih "Create a blank table".
  3. Beri nama tabel baru ini processingOrders.
  4. Hapus kolom default ("Notes", "Assignee", "Status") dan data contoh.
  5. Atur kolom-kolomnya agar persis cocok dengan output n8n Kita:
    • Ganti nama "Name" menjadi orderID (Tipe: Number).
    • Tambahkan kolom baru employeeName (Tipe: Single line text).

Tabel tujuan baru kita yang bersih, siap menerima data yang sudah difilter.

Langkah 5: Menyambungkan & Mengonfigurasi Ulang Airtable

Kembali ke n8n. Sekarang kita selesaikan alur barunya.

  1. Hubungkan output node Edit Fields ke input node Airtable.

Menyambungkan node yang sudah difilter ke Airtable.

  1. Buka node Airtable.
  2. Ubah parameter Table: ganti dari "orders" menjadi "processingOrders". (Anda mungkin perlu mengklik ikon 'Refresh List').

Mengarahkan node Airtable untuk mengirim data ke tabel baru Kita.

Pastikan Mapping Column Mode masih diatur ke Map automatically. Ini akan tetap berfungsi karena nama kolom Kita (orderID, employeeName) sama persis.

Langkah 6: Uji Coba Terakhir

Saatnya menguji! Klik Execute step pada node Airtable.

Buka tabel "processingOrders" Anda di Airtable. Anda akan melihat 14 baris data baru telah masuk, dan hanya 2 kolom yang terisi!

Sukses! Data yang bersih dan efisien telah disimpan.
